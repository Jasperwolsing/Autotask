# Set-AtwsAccountNote

## SYNOPSIS
This function sets parameters on the AccountNote specified by the -InputObject parameter or pipeline through the use of the Autotask Web Services API.
Any property of the AccountNote that is not marked as READ ONLY by Autotask can be speficied with a parameter.
You can specify multiple paramters.

## SYNTAX

### InputObject (Default)
```
Set-AtwsAccountNote [-WhatIf] [-Confirm] [<CommonParameters>]
```

### Input_Object
```
Set-AtwsAccountNote [-InputObject <AccountNote[]>] [-PassThru] [-AccountID <Int32>] [-ActionType <String>]
 [-AssignedResourceID <Int32>] [-ContactID <Int32>] [-EndDateTime <DateTime>] [-Name <String>] [-Note <String>]
 [-OpportunityID <Int32>] [-StartDateTime <DateTime>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### By_Id
```
Set-AtwsAccountNote [-Id <Int64[]>] [-AccountID <Int32>] [-ActionType <String>] [-AssignedResourceID <Int32>]
 [-ContactID <Int32>] [-EndDateTime <DateTime>] [-Name <String>] [-Note <String>] [-OpportunityID <Int32>]
 [-StartDateTime <DateTime>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### By_parameters
```
Set-AtwsAccountNote [-PassThru] -AccountID <Int32> -ActionType <String> -AssignedResourceID <Int32>
 [-ContactID <Int32>] -EndDateTime <DateTime> [-Name <String>] [-Note <String>] [-OpportunityID <Int32>]
 -StartDateTime <DateTime> [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
This function one or more objects of type \[Autotask.AccountNote\] as input.
You can pipe the objects to the function or pass them using the -InputObject parameter.
You specify the property you want to set and the value you want to set it to using parameters.
The function modifies all objects and updates the online data through the Autotask Web Services API.
The function supports all properties of an \[Autotask.AccountNote\] that can be updated through the Web Services API.
The function uses PowerShell parameter validation  and supports IntelliSense for selecting picklist values.

## EXAMPLES

### EXAMPLE 1
```
Set-AtwsAccountNote -InputObject $AccountNote [-ParameterName] [Parameter value]
Passes one or more [Autotask.AccountNote] object(s) as a variable to the function and sets the property by name 'ParameterName' on ALL the objects before they are passed to the Autotask Web Service API and updated.
```

### EXAMPLE 2
```
Same as the first example, but now the objects are passed to the funtion through the pipeline, not passed as a parameter. The end result is identical.
```

### EXAMPLE 3
```
Gets the instance with Id 0 directly from the Web Services API, modifies a parameter and updates Autotask. This approach works with all valid parameters for the Get function.
```

### EXAMPLE 4
```
Gets multiple instances by Id, modifies them all and updates Autotask.
```

### EXAMPLE 5
```
-PassThru
Gets multiple instances by Id, modifies them all, updates Autotask and returns the updated objects.
```
